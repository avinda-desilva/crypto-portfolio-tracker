## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Injection (XSS) | app/page.tsx — address render | mitigate | React JSX auto-escapes {wallet.address} text node; never set as href/src/style; no dangerouslySetInnerHTML; hard prohibition enforced in action block |
| T-02-05 | Injection (XSS) | app/page.tsx — error message render | accept | Error string is hardcoded in component ("Please enter a wallet address.") — not user-supplied; zero XSS surface |
| T-02-06 | Tampering | React state (wallet list) | accept | Client-side only; no server persistence in Phase 2; worst case user corrupts their own view |
| T-02-07 | Denial of Service | Address input (unbounded length) | accept | No server in Phase 2; long addresses stored only in React state; browser memory limit is the natural bound; add maxLength in Phase 12 if needed |
| T-02-08 | Information Disclosure | app/layout.tsx | accept | No secrets, no user data, no server-side rendering; pure HTML shell |
</threat_model>
