<objective>
Fix two critical bugs in the localStorage hooks added by plan 03-01:

**CR-01** — No runtime shape validation. `JSON.parse(raw) as Wallet[]` is a compile-time
assertion only. Valid JSON that is not an array (e.g. a JSON string, number, or object)
passes the try/catch and calls `setWallets` with a non-array, causing `wallets.map is not a
function` to crash the render. Fix: add `Array.isArray` + per-field type guard.

**CR-02** — Write-on-mount race. The sync effect fires after the *first* render when
`wallets` is still `[]`. React executes effects in declaration order: mount effect reads
localStorage and calls `setWallets(parsed)`, but the sync effect fires in the same flush
with the *pre-update* value `[]`, overwriting localStorage with an empty array before the
state update lands. Fix: gate the sync effect with a `useRef` flag set after the mount
effect completes.

Both bugs were confirmed by UAT test 3 (HUMAN-UAT.md): setting localStorage["wallets"] to
`not-json` caused a `wallets.map is not a function` crash on the first reload.
</objective>
