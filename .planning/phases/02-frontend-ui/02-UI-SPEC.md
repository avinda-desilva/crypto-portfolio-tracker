## Color

No custom palette. Browser default colors apply throughout.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #ffffff (browser default background) | Page background |
| Secondary (30%) | #f2f2f2 (browser default for `<table>` / grouped elements) | Wallet list rows |
| Accent (10%) | browser default blue (#0000ee / UA-dependent) | Submit button, focused inputs |
| Destructive | browser default red (#cc0000 / UA-dependent) | Delete button label text only |

Accent reserved for: the "Add Wallet" submit button and keyboard-focus ring on the address input and chain select. No other elements use accent color.

Destructive color reserved for: the delete control label (text "×" or "Remove") per wallet row. No confirmation dialog is required for this phase — single click removes immediately (KISS, per D-05 pattern).

**Source:** D-08 from 02-CONTEXT.md. All colors are browser UA defaults — no hex values need to be hardcoded in implementation.

## Interaction Contract

| Interaction | Trigger | Outcome |
|-------------|---------|---------|
| Add wallet | Click "Add Wallet" button or submit form | Validate non-empty address; if valid, append `{ address, chain }` to `useState<Wallet[]>` list; clear address input; keep chain selector at current value |
| Empty address submit | Click "Add Wallet" with blank address field | Display inline error "Please enter a wallet address." below the input; do not append to list |
| Remove wallet | Click "Remove" / "×" on a row | Remove that wallet from state immediately; no confirmation |
| Chain selector default | On initial render | Default selected option: "Ethereum" (value: `"ethereum"`) |

**Source:** D-03, D-04, D-05, D-07 from 02-CONTEXT.md; UI-01, UI-02, UI-03 from REQUIREMENTS.md.

## Accessibility Minimums

This phase uses plain HTML which provides baseline accessibility by default. The following must be maintained:

- Every `<input>` and `<select>` has an associated `<label>` via `htmlFor` / `id` pairing.
- The submit button is `type="submit"` so Enter key in the form submits correctly.
- The delete button is `type="button"` to prevent accidental form submission.
- No `aria-*` attributes are required beyond what semantic HTML provides.
